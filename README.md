# Point of Sale System with OCR Support

## Overview
This system is designed for a single-store minimart. Its purpose is to help the store manage daily operations in one place, from selling products and tracking stock to recording expenses, monitoring money movement, and handling GCash transactions.

The system should make selling faster and more organized for both Admin and Cashier users. It should also reduce manual errors by keeping product, sales, stock, and financial records clear and easy to review.

A key part of the system is OCR support. Instead of relying on barcode scanning in version 1, the system should help identify products through image-based text reading and product matching, while still allowing manual confirmation and manual search when needed.

## Core Goals
- help the minimart handle daily selling faster
- keep product and inventory records organized
- track sales clearly and accurately
- record expenses and other money movement in a practical way
- support GCash as a separate tracked balance from physical cash
- provide OCR-assisted product and transaction support
- allow offline selling when internet is unavailable
- preserve clear action history for important store activities

## What the System Should Do
### Product Management
The system should let Admin users create, edit, organize, and archive products. Products should include the details needed for selling, stock monitoring, and reporting.

### Inventory Tracking
The system should track stock changes clearly. It should support common stock actions such as adding stock, reducing stock, adjustments, damaged items, and lost items.

### Sales and POS
The system should support fast selling in the store. Users should be able to build a cart, choose a payment method, complete a sale, and review sales history. Important sales actions like edit, void, and refund should remain traceable.

### OCR Support
The system should support OCR for product lookup and for helping read GCash transaction details from images or screenshots. OCR should act as an assistant only, meaning the user should still confirm important results before they are finalized.

### GCash Tracking
The system should support one or more GCash accounts. It should record GCash-related transactions and keep them separate from the store’s physical cash records.

### Expenses and Money Movement
The system should record business expenses and other money-in or money-out activities. This helps the store monitor where money is coming from and where it is going.

### Dashboard and Reports
The system should give the store a simple overview of important information such as sales, expenses, stock alerts, money movement, GCash activity, and sync status.

### Offline Support
The system should still allow selling even when internet access is interrupted. Offline sales should be saved safely and synced later once the connection returns.

## Main Users
### Admin
The Admin should be able to manage the full system, including users, products, inventory, sales, expenses, money tracking, GCash, reports, and history.

### Cashier
The Cashier should focus mainly on selling-related tasks, especially product lookup, cart handling, payment recording, and other allowed sales actions.

## Key Characteristics
- built for one minimart only
- designed for fast daily operations
- made for tablet-first use, while still working on desktop
- focused on practical store needs instead of unnecessary complexity
- uses OCR support instead of barcode scanning in version 1
- keeps historical records visible and traceable
- supports both online and offline selling workflows

## Summary
The Point of Sale System with OCR Support is meant to be the main operating tool of the minimart. It should help the store sell faster, monitor stock better, track money more clearly, record GCash activity properly, and stay usable even during internet interruptions.
